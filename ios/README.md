# Am I Talking Too Much? — iOS

Native SwiftUI port of the Python/Streamlit conversation tracker. Calibrate
your voice once, then get live, color-coded feedback on how much of the
conversation's speaking time is yours. All analysis runs on-device; no audio
is stored or sent anywhere.

## Building

Requirements: Xcode 15+ and [XcodeGen](https://github.com/yonaskolb/XcodeGen)
(`brew install xcodegen`).

```bash
cd ios
xcodegen generate
open AmITalkingTooMuch.xcodeproj
```

Select a device or simulator and run. The microphone only works on a real
device or a simulator with mic input enabled.

## Running the tests

The signal-processing core is a standalone Swift package with no Apple
framework dependencies, so its tests run without the app — on macOS or
even Linux:

```bash
cd ios/VoiceCore
swift test
```

## Architecture

```
ios/
├── project.yml            XcodeGen spec (generates AmITalkingTooMuch.xcodeproj)
├── VoiceCore/             Platform-neutral SwiftPM package — all non-UI logic
│   ├── Sources/VoiceCore/
│   │   ├── MFCC.swift             framing, Hamming window, mel bank, DCT
│   │   ├── FFT.swift              pure-Swift radix-2 FFT, precomputed twiddles
│   │   ├── GaussianMixture.swift  diag-covariance GMM: EM training + scoring
│   │   ├── VoiceProfile.swift     Codable profile (Python-compatible JSON)
│   │   ├── VoiceMatcher.swift     calibration, match decisions, parity constants
│   │   ├── SampleSink.swift       thread-safe capture-queue → UI sample buffer
│   │   └── WavCodec.swift         16-bit PCM WAV encoder for playback
│   └── Tests/VoiceCoreTests/      parity tests against Python fixtures + unit tests
├── App/
│   ├── Sources/           SwiftUI app layer
│   │   ├── AudioCapture.swift         AVAudioEngine → 16 kHz mono stream, with
│   │   │                              route-change/interruption/media-reset recovery
│   │   ├── CalibrationView(-Model)    10 s voice recording, playback, save
│   │   ├── TrackingView(-Model)       live %, chart, stats, debug log
│   │   └── SettingsView.swift         re-calibrate, guide, privacy
│   └── Resources/PrivacyInfo.xcprivacy   no collection, no tracking
└── scripts/generate_fixtures.py   regenerates the parity fixtures from Python
```

### Parity with the Python app

`VoiceCore` is a numerical port of `voice_matcher.py`:

- **MFCC**: 512-sample frames, 256 hop, Hamming window, 26 mel filters,
  non-orthonormal DCT-II, 20 coefficients — identical to the Python
  pipeline (fixture tests agree to ~1e-14).
- **GMM scoring**: exact port of scikit-learn's
  `GaussianMixture.score_samples` for diagonal covariance, verified against
  sklearn outputs in the fixtures.
- **GMM training**: k-means++ + EM with a seeded PRNG. Training can't be
  bit-identical to sklearn (different RNG), so tests assert behavioral
  equivalence: the trained model accepts the calibration voice and rejects
  a different one, with a decisive margin.
- **Profile JSON**: `voice_profile.json` uses the same schema on both
  platforms (`weights`, `means`, `covariances`, `precisions_cholesky`,
  `threshold_score`), so a profile calibrated in the Streamlit app can be
  dropped into the iOS app's Documents directory and vice versa.
- **Decision logic**: 2-second chunks, speech gate at RMS 0.005, match gate
  at RMS 0.01, threshold = mean − 1.5·std of calibration scores, sigmoid
  confidence — all the same constants as the Python app.

### Test fixtures

`VoiceCore/Tests/VoiceCoreTests/Fixtures/*.json` are generated by running
the repository's **actual Python implementation** on deterministic synthetic
voices:

```bash
pip install numpy scipy scikit-learn
python3 ios/scripts/generate_fixtures.py
```

Regenerate them whenever `voice_matcher.py` changes, then run `swift test`
to confirm the port still agrees.

## Deliberate differences from the Python app

- **No microphone picker** — iOS routes input automatically (built-in mic,
  headset, Bluetooth).
- **No Whisper transcription / pyannote embeddings** — the optional Python
  extras are omitted to keep the app fully local and dependency-free. The
  GMM matcher is the sole speaker-ID path.
- **Background audio** is enabled so tracking continues with the screen
  locked during a meeting.
